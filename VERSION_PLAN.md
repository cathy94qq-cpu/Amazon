# WinWise 前端开发版本规划

> 当前 `index.html` / `styles.css` / `script.js` 中已经完成的用户端完整交互图，作为最终完整版本 / Target Version 保留。
>
> 本文档只用于前端开发范围拆分、版本标注和交付边界说明，不要求也不允许改变现有交互图中的页面结构、布局、样式、组件、文案和交互逻辑。

## 一、版本拆分总览表

| 版本号 | 版本目标 | 包含功能 | 不包含功能 | 依赖关系 | 前端开发重点 |
| --- | --- | --- | --- | --- | --- |
| V0.1 | 先完成最小可用 Chat 闭环，让用户可以进入聊天、发送问题、看到 AI 回复，并管理基础会话。 | Chat 首页 / 聊天主页面、基础聊天框、消息输入、消息发送、AI 回复展示、用户消息展示、会话新建、会话列表、会话切换、Chat 空状态 / 加载状态 / 错误状态。 | 附件上传、文件解析、Projects、Project 详情、Project 内 Chat、Library、Experts、账号订阅、搜索弹窗、分享、重命名、删除等非 Chat 核心增强能力。 | 无前置版本，是开发起点。 | 搭建 Chat 页面状态机、消息列表渲染、输入框交互、发送流程、基础会话数据结构、空 / 加载 / 错误反馈。 |

## 三、每个版本的交付范围说明

### V0.1 交付范围

#### 本版本目标

完成用户端最小可用 Chat 闭环。用户能够打开页面、理解这是 AI 问答系统、输入问题、发送消息、看到自己的问题和 AI 回复，并能在基础会话列表中切换会话。

#### 页面范围

- Chat 首页 / 新对话页。
- Chat 主聊天页面。
- 左侧基础会话列表。
- 精选示例入口。
- Chat 相关基础弹层或提示。

#### 功能范围

- 新建会话。
- 输入文本消息。
- 发送文本消息。
- 展示用户消息。
- 展示 AI 回复。
- 切换已有会话。
- 展示精选示例并进入对应问答。
- 基础会话列表渲染。

#### 状态范围

- Chat 首页默认状态。
- 问答页空状态。
- 消息发送中状态。
- AI 回复生成中状态。
- AI 回复成功状态。
- AI 回复失败状态。
- 会话列表为空状态。
- 会话切换加载状态。
- 网络错误提示。

#### 暂不开发内容

- 附件上传。
- 文件解析。
- 输入框中的附件展示。
- Projects。
- Project 详情。
- Project 内 Chat。
- Library / 资料库。
- Experts / 专家中心。
- 搜索弹窗。
- 账号管理、登录、订阅、账单。
- 会话更多菜单中的重命名、置顶、分享、删除。

#### 前端注意事项

- 不要把 Target Version 中的完整输入框功能全部搬进 V0.1。
- V0.1 的 Chat 输入框只做文本输入和发送。
- AI 回复可先使用 mock 数据或接口占位，但 UI 状态必须完整。
- 会话列表只需要满足新建、展示、切换，不需要复杂管理操作。
- 若后端接口未完成，前端需要封装清晰的数据适配层，避免后续 V0.2 / V0.3 大改。

### V0.2 交付范围

#### 本版本目标

在 V0.1 的 Chat 闭环上加入附件上传能力，让用户可以将文件与问题一起发送，并看到附件上传、解析、失败等完整反馈。

#### 页面范围

- V0.1 的 Chat 首页 / Chat 主页面。
- Chat 输入框附件区域。
- 消息列表中的附件展示区域。
- 附件相关提示 / 错误状态。

#### 功能范围

- Chat 输入框附件入口。
- 文件选择。
- 文件类型 / 大小校验。
- 附件展示。
- 上传中状态。
- 上传成功状态。
- 上传失败状态。
- 删除附件。
- 附件随消息发送。
- 附件解析 / 处理中状态。
- 附件解析失败提示。

#### 状态范围

- 未选择附件状态。
- 已选择附件状态。
- 上传中状态。
- 上传成功状态。
- 上传失败状态。
- 删除附件成功状态。
- 附件解析中状态。
- 附件解析成功状态。
- 附件解析失败状态。
- 附件随消息发送失败状态。

#### 暂不开发内容

- Projects。
- Project 管理。
- Project 详情页。
- Project 内文件管理。
- Library 独立资料库。
- Experts。
- 账号管理、订阅、账单。
- 文件在资料库中的管理、下载、删除。
- Project 文件 / 知识上下文入口。

#### 前端注意事项

- V0.2 的附件只属于 Chat 消息上下文，不等同于 Library 文件管理。
- 上传入口、上传状态、附件随消息发送必须与消息发送状态解耦，避免一个失败导致全部状态混乱。
- 附件解析中需要给用户明确反馈，避免用户误以为系统卡住。
- 文件类型、大小限制需要在前端先拦截，并给出友好错误提示。

### V0.3 交付范围

#### 本版本目标

在 V0.2 基础上补齐完整用户端功能模块，包括 Projects、Library、Experts、搜索、账号订阅等，使实现接近当前 Target Version。

#### 页面范围

- Projects 列表页。
- Project 详情页。
- Project 内 Chat。
- Project 相关弹窗：新建、设置、分享、删除。
- Library / 资料库页面。
- Experts / 专家中心页面。
- 搜索弹窗。
- 账号登录与账户菜单。
- 账户管理弹窗。
- 升级计划与订阅相关页面 / 弹窗。
- 精选示例探索更多页面。
- 会话更多操作相关弹窗。

#### 功能范围

- Projects 列表展示。
- 新建 Project。
- Project 详情页展示。
- Project 内 Chat。
- Project 内聊天记录。
- Project 相关文件 / 知识上下文入口。
- Project 空 / 加载 / 错误状态。
- Library 文件列表、tab、上传、下载、删除、空状态。
- Experts 列表、分类筛选、专家详情、召唤专家。
- 搜索会话和项目。
- 账号登录、账户菜单、账户管理。
- 订阅计划、订阅成功、账单、席位管理。
- 会话重命名、置顶、分享、删除。

#### 状态范围

- Project 列表为空状态。
- Project 创建中 / 成功 / 失败状态。
- Project 详情加载状态。
- Project 内 Chat 空状态。
- Project 内 Chat 发送中 / 成功 / 失败状态。
- Library 文件为空状态。
- Library tab 无结果状态。
- Library 上传中 / 成功 / 失败状态。
- Library 删除确认状态。
- Experts 加载 / 空 / 错误状态。
- 搜索无结果状态。
- 登录失效 / 权限不足状态。
- 订阅成功 / 账单为空 / 席位邀请状态。

#### 暂不开发内容

- 后台管理端。
- 真实支付结算后端。
- 真实 SSO / MFA。
- 真实大文件解析服务。
- 生产级权限策略。
- 复杂团队组织管理。

#### 前端注意事项

- V0.3 是 Target Version 的完整模块阶段，不应反向影响 V0.1 / V0.2 的基础 Chat 能力。
- Experts 的“召唤专家”只影响新对话输入框，不应污染历史会话输入框。
- Library 的上传文件是资料库文件管理，不要与 V0.2 Chat 附件上传混淆。
- Project 内 Chat 与全局 Chat 需要明确区分数据上下文。
- 账号 / 订阅模块如无后端接口，可先按静态状态和 mock 数据实现。

## 四、在现有交互图基础上增加版本标注说明

### 标注原则

- 不改变现有页面结构。
- 不改变现有布局、样式、组件、文案和交互逻辑。
- 不移动、不删除、不重构现有任何页面或组件。
- 只允许在交互图旁边增加版本标签、说明页或开发范围说明。
- 当前完整交互图保留为 Target Version。

### 推荐新增标注

1. 在 Chat 首页 / Chat 主页面旁增加 `V0.1` 标签。
   - 标注说明：基础 Chat 闭环。
   - 覆盖：文本输入、发送、消息展示、会话新建、会话切换。

2. 在 Chat 输入框附件入口旁增加 `V0.2` 标签。
   - 标注说明：附件上传能力。
   - 覆盖：文件选择、附件状态、附件随消息发送、解析中状态。

3. 在 Projects、Library、Experts、Account、订阅、搜索等模块旁增加 `V0.3` 标签。
   - 标注说明：完整功能模块。
   - 覆盖：非 Chat 核心能力、复杂弹窗、跨模块操作。

4. 新增一个“版本规划说明”页面。
   - 内容包括：V0.1 / V0.2 / V0.3 的目标、包含范围、不包含范围、依赖关系。
   - 该页面不与现有用户流程相连，作为开发说明使用。

5. 新增一个“开发范围说明”页面。
   - 用表格列出页面与版本归属。
   - 用醒目说明强调：Target Version 不等于 V0.1 全部要开发。

### 建议标签文案

- `V0.1 Chat 基础能力`
- `V0.2 Chat + 附件`
- `V0.3 Projects / Library / Experts / Account`
- `Target Version 完整交互图`

## 六、前端落地建议

1. 将当前完整交互图命名为 `Target Version`。
2. 前端开发不要以 Target Version 的所有功能作为 V0.1 范围。
3. 先实现 V0.1 的 Chat 数据结构与消息状态，后续 V0.2 / V0.3 复用。
4. V0.2 只扩展 Chat 输入框和消息发送链路，不引入 Project 或 Library 概念。
5. V0.3 再引入 Projects、Library、Experts、Account 等模块。
6. 每个版本都需要有明确的“不开发范围”，避免需求自然外溢。
